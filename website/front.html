    Each input segment is wrapped in a cryptographically generated per-session UUID tag. Attackers can't pre-craft escape sequences — they don't know the delimiter. Boundary violations are detected instantly at parse time, with no ML inference required.   

  
    
      
Integration
      

Three lines. Done.

      

        Drop LightShield into any existing pipeline. Works with OpenAI, Anthropic, local models — anything that takes a string prompt.       

    
    
      
        
          
          
          
        
        rag_pipeline.py         python       
      
from lightshield import Shield

# Initialize once per session
shield = Shield()

# Wrap your inputs
prompt = shield.wrap(
    system="You are a helpful assistant.",
    user=user_query,
    retrieved=docs_from_vector_db
)

# Call your LLM as normal
response = llm(prompt)

# Validate on the way out
safe = shield.validate(response)

Why LightShield
  

Security as architecture,
not content moderation

  
    
      
⚡
      
Near-zero latency
      
Pure string manipulation at parse time. No ML inference, no network call, no GPU. Microseconds, not milliseconds.
    
    
      
🔑
      
Per-session UUID encapsulation
      
Each session gets a fresh cryptographic UUID as its delimiter. Attackers can't pre-craft escape sequences — they can't know the boundary.
    
    
      
🧩
      
Framework-agnostic
      
Works with any LLM — OpenAI, Anthropic, Mistral, local models. No lock-in. Plug it into existing pipelines in minutes.
    
    
      
🛡
      
Two-layer defense
      
Structural prevention at input + output-side validation on response. If boundary compliance fails at inference, the validator catches it.
    
    
      
🤖
      
Agentic-safe
      
Protects pipelines where a successful injection isn't just annoying — it's dangerous. Prevents tool manipulation from malicious retrieved content.
    
    
      
📦
      
Lightweight by design
      
Built for the two-person startup shipping an AI feature — not enterprises who can afford NeMo. Zero dependencies beyond the standard library.
